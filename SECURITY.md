# Clinical Accuracy and Security Policy

## Reporting Clinical Content Errors

**Clinical accuracy is our highest priority.** If you find information that is
incorrect, outdated, or could mislead a nurse or DCSO about clinical safety practice:

1. **Open a [Clinical Content Error issue](../../issues/new?template=clinical-content-error.md)**
2. Cite the authoritative UK source (BNF, NICE, NMC, NHS England, MHRA)
3. Mark urgency: 🔴 Urgent / 🟡 Important / 🟢 Minor

Clinical content errors are reviewed as a priority and corrected in the next release.

## Reporting Translation Errors

If you find a mistranslation in any of the multilingual glossary files:

1. Open an issue using the "Clinical Content Error" template
2. State the language and the correct translation
3. If possible, confirm you are a native speaker or clinical professional in that language

All translations are currently AI-generated and **have not been independently verified
by native-speaking clinical professionals**. We actively seek native speaker reviewers.

## Scope of This Policy

This repository contains **educational content only** — not software that processes
patient data, not a medical device, and not a clinical decision support system.

There are therefore no software security vulnerabilities to report in the traditional
sense. The primary "security" concern for this resource is **clinical accuracy**.

## What This Repository Is NOT

- Not a live system (no servers, no APIs, no patient data)
- Not affiliated with NHS England, NMC, NICE, or MHRA
- Not a substitute for formal DCSO training or qualified clinical safety advice
